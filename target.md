# Cesium 单页大屏可视化项目开发目标说明（target.md）

## 1. 项目概述

本项目旨在构建一个基于 **Vue 3 + TypeScript + Element Plus + Cesium + ECharts** 的单页大屏可视化系统，并配套 **Django + PostGIS** 后端，实现矿区空间数据的统一展示、联动分析与基础统计。

项目整体仅保留 **一个主页面**，页面中心为 Cesium 三维地球/三维场景，用于展示各类地理空间数据；页面左右两侧为统计面板、图层控制、对象详情与图表区域。系统重点在于“展示 + 联动 + 查询”，暂不以复杂业务管理为目标。

本阶段优先实现核心功能，围绕以下四类数据展开：

1. **钻孔点位置数据**
2. **钻孔分层数据（Excel）**
3. **SHP 边界数据（矿区边界、工作面边界）**
4. **TIFF 栅格成果图（沉降图、热力图、专题图等）**

---

## 2. 项目目标

### 2.1 总体目标

构建一个可运行、可演示、具备空间展示和基础分析能力的 Cesium 大屏项目，能够直观展示矿区相关空间数据，并实现地图、图表、属性面板之间的联动。

### 2.2 第一阶段核心目标

第一版仅实现最核心、最有展示价值的功能，不引入复杂权限体系、流程审批、多角色管理等内容。

核心目标包括：

- 完成单页大屏布局搭建
- 完成 Cesium 三维场景初始化与基础控件封装
- 完成钻孔点、钻孔分层、SHP 边界、TIFF 图层的加载与展示
- 完成左侧统计面板与右侧详情面板
- 完成 ECharts 图表展示与地图联动
- 完成基础图层开关、透明度控制、飞行定位、点击查询等交互
- 完成 Django + PostGIS 数据存储与基础接口服务

---

## 3. 页面定位与展示结构

项目为 **单页大屏**，整体布局建议为：

- **顶部**：系统标题栏、当前时间、数据状态提示
- **左侧面板**：图层控制、全局统计信息、部分图表
- **中间主区域**：Cesium 三维场景
- **右侧面板**：当前选中对象详情、钻孔分层信息、图例、补充图表
- **底部状态栏（可选）**：经纬度、高程、当前图层说明、系统状态

布局应为典型深色大屏风格，突出地图主体，左右卡片采用可折叠或滚动式设计。

## 5. 数据内容与业务对象设计

## 5.1 钻孔点数据

用于展示钻孔的空间位置与基础属性。

建议字段：

- id
- borehole_code（钻孔编号）
- name（钻孔名称）
- longitude
- latitude
- elevation（可选）
- depth_total（总深度，可选）
- workface_name（所属工作面，可选）
- remark（备注，可选）
- geom（PostGIS Point）

前端表现形式：

- Cesium 点标记 / billboard
- 支持点击高亮
- 支持飞行定位
- 支持右侧详情展示

## 5.2 钻孔分层数据（Excel）

Excel 中记录每个钻孔的分层信息。推荐后端读取 Excel 后入库，前端通过接口按钻孔查询。

建议 Excel 字段：

- 钻孔名称
- 地层名称
- 深度
- 厚度

如一层一行，则后端需要按钻孔分组整理。

建议数据库字段：

- id
- borehole_id（外键）
- layer_name
- top_depth
- thickness
- bottom_depth（可由后端计算）
- color（可由后端或前端映射） 
- sort_order

前端表现形式：

- 点击钻孔后，在右侧面板展示分层表格
- 展示分层柱状示意图或简化分层条带图
- 支持统计某类岩层出现次数、累计厚度等信息

> 第一阶段不强制要求在 Cesium 中直接渲染真实三维分层柱体，但建议预留接口与组件结构，后续可以扩展为三维钻孔柱体。

## 5.3 SHP 边界数据

用于展示矿区边界、工作面边界等面状或线状对象。

建议分类：

- 矿区边界
- 工作面边界
- 其他功能区边界（可选）

建议字段：

- id
- name
- type（mine / workface / region）
- area（可选）
- perimeter（可选）
- properties（JSON，保留原始属性）
- geom（PostGIS Polygon / MultiPolygon / LineString）

前端表现形式：

- 面对象用半透明填充 + 发光边界线
- 支持显隐控制
- 支持点击高亮
- 支持飞行至区域
- 点击后右侧展示属性

## 5.4 TIFF 栅格成果图

用于展示各种处理好的空间成果图，如：

- InSAR 沉降图
- 热力图
- 风险分区图
- 属性反演图
- 其他专题影像

建议字段：

- id
- name
- type
- file_path / url
- bounds
- opacity
- legend_config（JSON）
- description
- time_tag（可选）

前端表现形式：

- 作为 Cesium 影像叠加层显示
- 支持图层开关
- 支持透明度调节
- 支持图例展示
- 支持切换不同专题图

> 如果 TIFF 很大，建议后端预处理切片或转换服务形式，不建议前端直接硬读超大原始栅格。

## 6. 核心功能范围

## 6.1 必须实现的功能

### 6.1.1 大屏基础框架

- 实现单页大屏布局
- 实现顶部标题栏
- 实现左中右三区域
- 整体风格统一为深色科技风
- 页面自适应 16:9 大屏比例

### 6.1.2 Cesium 场景初始化

- 初始化 Cesium Viewer
- 配置深色底图或影像底图
- 关闭不必要默认控件，保留必要交互
- 设置初始视角飞到矿区范围
- 支持基础相机操作（缩放、旋转、俯仰）

### 6.1.3 钻孔点展示

- 从后端获取钻孔点数据
- 在 Cesium 中渲染钻孔点
- 支持点击选中
- 支持 hover 提示（简要信息）
- 支持点击后右侧展示详情
- 支持通过列表点击后飞行至目标点位

### 6.1.4 钻孔分层信息展示

- 支持按钻孔查询分层数据
- 右侧面板显示分层表格
- 右侧显示分层柱状示意图或颜色条带图
- 统计钻孔总深度、分层数量、主要层位信息

### 6.1.5 SHP 边界展示

- 支持矿区边界加载
- 支持工作面边界加载
- 支持不同边界分类样式区分
- 支持点击高亮与属性展示
- 支持飞行至边界范围

### 6.1.6 TIFF 图层叠加

- 支持专题 TIFF 图层加载
- 支持透明度调节
- 支持图层开关
- 支持图例展示
- 支持切换不同专题图

### 6.1.7 图层控制

- 提供图层管理面板
- 支持以下图层开关：
  - 钻孔点图层
  - 矿区边界图层
  - 工作面边界图层
  - TIFF 专题图图层
- 支持 TIFF 图层透明度调节
- 支持快速定位到某类图层范围

### 6.1.8 左侧统计面板

建议至少包含以下内容：

- 钻孔总数
- 工作面数量
- 当前加载专题图数量
- 边界对象数量
- 钻孔分层统计信息（如出现频次前几名）

### 6.1.9 ECharts 图表

建议第一版至少实现 3 个图表：

1. **钻孔分层类型占比图**（饼图）
2. **各工作面钻孔数量统计图**（柱状图）
3. **钻孔深度分布图**（柱状图或区间分布图）

要求：

- 图表数据来自后端接口
- 地图选中对象后，部分图表可联动更新
- 图表点击后可触发地图筛选或定位（至少预留接口）

### 6.1.10 右侧详情面板

右侧内容应根据当前选中对象动态变化：

#### 选中钻孔时

展示：

- 钻孔编号/名称
- 坐标
- 总深度
- 所属区域/工作面
- 分层列表
- 分层示意图

#### 选中边界时

展示：

- 名称
- 类型
- 面积/周长（如有）
- 附加属性
- 区域内钻孔数量（可后端统计）

#### 选中 TIFF 图层时

展示：

- 图层名称
- 类型
- 时间/说明
- 图例
- 当前透明度

## 7. 第一阶段不做的内容

为了保证项目尽快落地，以下内容暂不纳入第一版：

- 用户登录/权限管理
- 多页面后台管理系统
- 在线编辑 SHP / 钻孔数据
- 复杂三维地质实体建模
- 真正的三维钻孔地层体渲染
- 实时监测流数据
- 多人协作
- 工作流审批
- 完整文件上传中心
- 复杂时序动画系统

如果需要导入数据，第一阶段可采用以下任一方式：

- Django 后台管理页面上传
- 本地脚本导入数据库
- 简单接口上传（仅管理员使用）

## 8. 可扩展功能建议（仅预留，不要求本阶段完成）

## 11. 接口设计建议

以下为建议接口，不要求命名完全一致，但应保持清晰。

## 11.1 钻孔相关

### 获取钻孔列表

`GET /api/boreholes/`

支持参数：

- `keyword`
- `workface`
- `page`
- `page_size`

### 获取钻孔 GeoJSON

`GET /api/boreholes/geojson/`

返回钻孔点空间数据，用于 Cesium 加载。

### 获取单个钻孔详情

`GET /api/boreholes/{id}/`

### 获取单个钻孔分层信息

`GET /api/boreholes/{id}/layers/`

---

## 11.2 边界相关

### 获取边界列表

`GET /api/boundaries/`

支持参数：

- `type=mine|workface|region`

### 获取边界 GeoJSON

`GET /api/boundaries/geojson/`

### 获取单个边界详情

`GET /api/boundaries/{id}/`

---

## 11.3 栅格图层相关

### 获取专题图列表

`GET /api/rasters/`

### 获取单个专题图详情

`GET /api/rasters/{id}/`

返回内容建议包括：

- 名称
- 类型
- 服务地址 / 图片地址
- 边界范围
- 图例配置
- 默认透明度

---

## 11.4 大屏统计接口

### 获取首页总览统计

`GET /api/dashboard/overview/`

返回：

- 钻孔总数
- 工作面总数
- 边界数量
- 栅格图层数量

### 获取分层占比统计

`GET /api/dashboard/layer-distribution/`

### 获取工作面钻孔数量统计

`GET /api/dashboard/workface-boreholes/`

### 获取钻孔深度分布统计

`GET /api/dashboard/borehole-depth-distribution/`

## 14. Cesium 实现建议

## 14.1 场景配置

建议：

- 隐藏默认无关控件
- 保留必要交互
- 初始定位到矿区范围
- 开启深色风格适配
- 统一管理数据源与图层对象

## 14.2 数据加载方式

### 钻孔点

- 后端返回 GeoJSON
- 前端转换为 Cesium Entity 或 DataSource

### 边界

- 后端返回 GeoJSON
- 前端以 Polygon / Polyline 方式渲染

### TIFF

- 若已有可直接叠加的图像边界与服务地址，则按影像图层加载
- 若后端输出切片服务，可按 URL 模板加载

## 14.3 交互要求

- 点击地图对象时可识别对象类型
- 不同类型对象走不同详情渲染逻辑
- 支持飞行到点或区域
- 支持 hover 基础提示

---

## 15. ECharts 实现建议

图表风格应与深色大屏保持一致。

建议统一封装 ECharts 基础组件，支持：

- 自适应 resize
- 深色主题
- 通用 tooltip
- 图例样式统一

第一版图表重点是展示价值，不追求过度复杂交互。

---

## 16. PostGIS 设计建议

数据库中建议使用 PostGIS 几何类型保存空间对象：

- 钻孔：`Point`
- 边界：`Polygon / MultiPolygon / LineString`
- 栅格：可只保存元信息，不强制直接存储 raster 本体

注意事项：

- 所有空间数据应统一坐标系
- 导入 SHP 时要检查投影并做转换
- 前端 Cesium 通常使用经纬度坐标，后端返回时需明确坐标参考系统
- 若 TIFF 有投影信息，需确保与场景坐标对齐

4. 后端模型与接口命名保持统一、清晰。
5. 优先保证：能展示、能联动、能演示，而不是先做复杂优化。
6. 对于 SHP / TIFF / Excel 的导入，优先采用简单可控的方案，不要一开始就引入过重的服务架构。
7. 所有空间数据必须注意坐标系统一。
8. 保留后续扩展三维钻孔柱体、时间轴专题图、剖面分析的空间。

## 23. 补充说明

如果执行过程中需要在“直接前端解析文件”与“后端入库后提供接口”之间做取舍，第一版推荐优先采用：

- **钻孔点 / 钻孔分层 / 边界：后端入库 + API 提供**
- **TIFF：数据库保存元信息 + 静态资源或服务地址提供前端加载**
